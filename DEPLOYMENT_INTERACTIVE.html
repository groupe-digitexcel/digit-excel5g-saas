Contenu du schéma (résumé)
-- Tables créées:
CREATE TABLE public.profiles (...)  -- extension de auth.users
CREATE TABLE public.ai_jobs (...)    -- historique créations IA

CREATE TRIGGER on_auth_user_created  -- +100 crédits à l'inscription

ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;

CREATE FUNCTION deduct_credits(user_id, amount) RETURNS BOOLEAN

-- Storage bucket policies:
-- Public read + Authenticated write pour ai-outputs

ALTER TABLE public.ai_jobs

🔧 Dépannage

❌Page blanche ou erreur 500 au démarrage
▶
Vérifier que toutes les variables d'environnement sont dans Vercel Settings → Environment Variables. La cause la plus fréquente: NEXT_PUBLIC_SUPABASE_URL manquante. Ajouter et redéployer.
❌"Missing Supabase environment variables"
▶
Les variables NEXT_PUBLIC_SUPABASE_URL et NEXT_PUBLIC_SUPABASE_ANON_KEY ne sont pas définies dans Vercel. Aller dans Settings → Environment Variables, les ajouter pour tous les environnements et redéployer.
❌Inscription fonctionne mais redirige en boucle
▶
Vérifier que l'URL Vercel est dans Supabase → Authentication → URL Configuration → Redirect URLs. Souvent oublié. Ajouter https://votre-app.vercel.app/auth/callback.
❌Génération d'images échoue (erreur 500)
▶
Vérifier REPLICATE_API_KEY dans Vercel — elle doit commencer par r8_. Vérifier les logs Vercel → Functions → api/ai/image pour le message d'erreur exact. Vérifier aussi que le compte Replicate a des crédits.
❌Flyers/Chansons retournent seulement le contenu de secours
▶
Vérifier OPENROUTER_API_KEY dans Vercel — elle doit commencer par sk-or-. Ajouter $5 de crédit sur openrouter.ai. Vérifier les logs Vercel pour les erreurs API.
❌"Crédits insuffisants" avec des crédits disponibles
▶
La fonction RPC deduct_credits n'a pas été créée. Ré-exécuter le schéma SQL complet dans Supabase SQL Editor. Vérifier dans Supabase → Database → Functions que deduct_credits existe.
❌Panel Admin non accessible (/admin redirige)
▶
Vérifier que votre rôle est bien 'admin' dans la table profiles. Supabase → SQL Editor: SELECT email, role FROM public.profiles WHERE email = 'votre@email.com';
❌git push: "remote: Permission denied"
▶
Utiliser un Personal Access Token au lieu de votre mot de passe. GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token → cocher scope "repo" → utiliser le token comme mot de passe.
❌Images IA générées non affichées après génération
▶
Vérifier que le bucket Supabase ai-outputs existe et est en mode public. Vérifier que les policies ont bien été créées par le schéma SQL. Dans Supabase → Storage, l'URL publique du bucket doit être accessible.
